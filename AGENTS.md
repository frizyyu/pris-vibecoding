# AGENTS

## Цель проекта

Прототип веб-приложения для управления задачами и дедлайнами (MVP), разработанный с помощью средств вайб-кодинга для лабораторной работы.

## Стек

- Next.js
- TypeScript
- Tailwind CSS
- Prisma
- SQLite

## Принципы разработки

- Не усложнять архитектуру: это MVP, а не production-система.
- Использовать минимально достаточную декомпозицию.
- Не подключать тяжёлые библиотеки без необходимости.
- Избегать дублирования кода.
- Не оставлять TODO и незавершённые куски.

## Правила работы агента

- После значимых изменений всегда проверять:
  - установку зависимостей;
  - сборку проекта;
  - запуск dev-сервера.
- Исправлять ошибки сразу, а не откладывать.
- Не останавливаться на полуреализации.
- Обновлять README при изменениях.

## Требования к качеству

- Код должен быть читаемым.
- Логика должна быть понятной.
- Проект должен быть пригоден для анализа в лабораторной работе.
